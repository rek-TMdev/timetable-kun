時間割くん HTMLマニュアル

時間割くんは、履修選択と時間割候補の確認を支援する Windows 向けデスクトップアプリ群です。このページでは、本体、ドライバ、チェッカーの基本操作と注意点を一つにまとめています。

構成

リポジトリには、通常利用者向けの本体アプリ、設定編集用のドライバ、Excel 時間割の検証用チェッカーが含まれます。設定ファイルは JSON、時間割入出力は Excel または専用形式ファイルを使います。

本体

設定ファイルを読み込み、課程やコースなどの階層区分ごとに時間割候補を生成します。検索、枠指定、固定科目、候補保存、Excel 出力を扱います。

ドライバ

設定 JSON を編集します。科目マスタ、別名、履修条件、枠レイアウト、保存位置、一般設定を画面から調整します。

チェッカー

保存済みの Excel 時間割を読み込み、単位数、必須科目、前提科目などを検証します。Dry Run、結果コピー、CSV レポートに対応します。

起動方法

ソースから起動する場合

Windows と PowerShell を前提に、仮想環境を作成して依存関係を入れます。

py -3.12 -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt

各アプリは次のコマンドで起動できます。

python src\app\time_manager_main.py
python src\driver\config_editor_main.py
python src\checker\timetable_checker_main.py

配布版を使う場合

配布版では、利用者向けのランチャーを起動します。ランチャーはスプラッシュ画面を表示し、内部の本体実行ファイルが準備できたら自動で閉じます。

用途 起動するもの 主な利用者
時間割作成 本体アプリのランチャー 時間割候補を作成・確認する人
設定編集 ドライバのランチャー 設定 JSON を管理する人
時間割検証 チェッカーのランチャー Excel 時間割をまとめて確認する人

サンプル設定の選び方

初回起動には設定 JSON が必要です。examples/sample_config.json には、架空データのみで構成したサンプル設定が入っています。

  1. examples/sample_config.json を選択します。
  2. 必要に応じて編集用の作業コピーを作成します。
  3. 科目、階層区分、枠レイアウト、保存位置、履修条件を用途に合わせて編集します。
  4. 本体またはチェッカー起動時の設定選択ダイアログで、その JSON を選びます。
  5. 候補が複数ある場合は、表示名と更新日を確認して選択します。
サンプル内容: 平日5日・1日6限の枠に、課程 として 全日制 を用意しています。全日制 にはアンカーを設定しています。設定内の葉キーは 全日制_2年 と 全日制_3年、枠名は A1 や X1 などのスロット ID です。芸術科目セレクト用の候補は 書道Ⅰ、美術Ⅰ、音楽Ⅰ です。

本体とチェッカーは、起動時にアプリ配置先と現在の作業ディレクトリから JSON を探します。候補がない場合は、ファイル選択ダイアログから手動で指定できます。専用形式ファイル .tm.json とアプリ個人設定用の config.json は設定候補から除外されます。

本体マニュアル

基本の流れ

  1. 本体を起動し、使用する設定 JSON を選択します。
  2. 課程やコースなどの階層がある場合は、対象範囲を選びます。
  3. 左側のリストから作業する区分タブを開きます。
  4. 科目を選択し、必要に応じて検索欄や「枠で検索」を使って候補を絞ります。
  5. 固定したい枠がある場合は固定設定を使い、候補生成から外したい枠を調整します。
  6. 「組み合わせを絞り込み」で科目数、単位数、固定科目を含めるかなどを確認します。
  7. 「実行」で時間割候補を生成し、内容を確認して保存します。

主な機能

  • 階層区分ごとの時間割タブを自動作成します。
  • 科目名検索、枠指定検索、候補の絞り込みに対応します。
  • 前提科目、同時選択不可、必須科目、科目数、単位数の条件を設定に基づいて扱います。
  • 固定枠や手動追加した科目と候補の競合を視覚的に確認できます。
  • 複数のプロファイルを作成し、候補や固定状態を分けて保存できます。
  • Excel への保存と、専用形式ファイル .tm.json への保存に対応します。
  • 保存済み Excel または専用形式ファイルを読み込み、作業を再開できます。

注意事項

  • 候補が出ない場合は、同時選択不可、必須科目、単位数の条件が厳しすぎないか確認します。
  • Excel 出力先のセル位置は設定 JSON の保存位置設定に依存します。出力前にドライバで確認してください。
  • 専用形式ファイルには作業状態が保存されます。
  • 自動選択された前回設定が意図と違う場合は、「設定ファイルを変更」から選び直します。

ドライバマニュアル

基本の流れ

  1. ドライバを起動し、「ファイルを開く」から設定 JSON を選択します。
  2. 左側のタブで編集対象を切り替えます。
  3. 科目マスタ、別名、レイアウト、履修条件、保存位置などを編集します。
  4. 必要に応じて Excel から設定を取り込み、または Excel へ設定を書き出します。
  5. 保存前に内容を確認し、「上書き保存」または「名前を付けて保存」を実行します。

編集できる主な項目

タブ 内容
科目詳細 区分ごとの単位数、科目番号、枠グループ、特別単位数を設定します。
科目マスタ 利用可能な科目名を追加、編集、削除、並べ替えします。
別名 表記ゆれを標準科目へ紐づけます。自動生成機能も利用できます。
スロットレイアウト 時間割表の行列と枠名を設定します。
必須科目 指定範囲で満たすべき科目条件を設定します。
前提科目 ある科目を選ぶために必要な科目を設定します。
同時選択不可 同じグループ内で同時に選べない科目を設定します。
保存位置 Excel 出力時に各枠を書き込むセル位置を設定します。
区分設定 課程やコースなどの階層、選択時の区切りを設定します。
一般設定 フォント、候補生成数、テーマ、共有設定の参照先などを設定します。

統合と競合解決

「設定ファイルを統合」では、基準、手元、外部の差分を比較して、可能な部分を自動統合します。表レイアウトや値が衝突した部分は競合として表示され、どちらを採用するか選べます。

編集時の注意: 共有元として使う設定を直接編集すると、利用者全体に影響します。変更内容を確認しやすいよう、作業コピーを分けて扱います。

チェッカーマニュアル

基本の流れ

  1. チェッカーを起動し、検証に使う設定 JSON を選択します。
  2. 必要に応じて階層選択ボタンから対象範囲を切り替えます。
  3. 「時間割ファイルを選択」で Excel ファイルを選ぶか、画面へドラッグアンドドロップします。
  4. 最小単位数を確認し、必要なら値を変更します。
  5. 検証結果サマリー、詳細、移動予定を確認します。
  6. 必要に応じて CSV レポートを出力します。

検証内容

  • 設定されたセル位置から Excel の科目情報を読み取れるか。
  • 読み取った科目と固定科目の合計単位数が下限を満たすか。
  • 必須科目条件を満たすか。
  • 前提科目が不足していないか。
  • 読み込みエラーやセル位置の不整合がないか。

Dry Run とファイル移動

チェッカーは初期状態で Dry Run が有効です。この状態では不合格ファイルを移動せず、移動予定と詳細だけを表示します。Dry Run を外すと、不合格ファイルを同じフォルダ内の分類用フォルダへ移動します。

安全な使い方: 最初は Dry Run のまま検証し、詳細を確認してからファイル移動を実行してください。大量の Excel を扱う場合は、事前にバックアップを作成します。

ビルド定義

PyInstaller と Inno Setup 用の定義が含まれます。ビルド定義は、リポジトリ内のファイルを相対パスで参照します。

pyinstaller installer\時間割くん.spec
pyinstaller installer\時間割くんツール.spec
定義 内容
installer\時間割くん.spec 本体アプリと本体ランチャーをまとめる PyInstaller 定義です。
installer\時間割くんツール.spec ドライバ、チェッカー、それぞれのランチャーをまとめる PyInstaller 定義です。
installer\時間割くん.iss 本体アプリ用の Inno Setup 定義です。
installer\時間割くんツール.iss ドライバとチェッカー用の Inno Setup 定義です。
